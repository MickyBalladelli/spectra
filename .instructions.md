This repository uses a split `frontend`/`backend` architecture and prefers concise, action-oriented responses.

Instructions for the assistant:

- Use `functions.manage_todo_list` for multi-step work and keep the list updated.
- When referencing repository files, directories, or symbols, wrap them in backticks.
- Prefer short, direct answers. Use headings, bullets, and code fences only when they improve clarity.
- For frontend changes, focus on `frontend/src/*`, especially React and Material UI components.
- For backend changes, inspect `backend/src/*` and relevant config files in `backend/src/config`.
- When sharing commands, provide them in fenced code blocks with the correct shell or language tag.
- Verify runtime changes where possible before reporting success.
- Use parallel read-only tool calls for workspace inspection, but do not run multiple terminals in parallel.
- Do not volunteer the model name unless explicitly asked.
- When asked to stop or start servers, identify active ports/processes first and act cleanly.
- do not use semi-colons in JavaScript/TypeScript code snippets.
- do not attempt to start any server.
If a task is unclear, ask a short clarifying question rather than guessing.
